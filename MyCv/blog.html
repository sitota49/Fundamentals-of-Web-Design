Absolute Size Units Vs. Relative Size Units

CSS provides different length units to describe properties like margin, padding, width, height and font size. Some of these length units include %, px, rem and em. These length units can be grouped into two namely absolute length units and relative length units.

Absolute length units are used to express fixed lengths like in cm, mm and pxs. They are not recommended for screen use because screen size differs from device to device.

Relative length units specify length in relative to other length units. Using relative length units is recommended for better scaling results while rendering our web page on different devices. Examples of relative length units include em(relative size to current element), rem(relative size to root element) and %(relative to parent element) The Web Content Accessibility Guidelines published by W3C states that:

  • “Users must be able to resize text without assistive technology up to 200 percent, without loss of content or functionality.” –WCAG 1.4.4
  • “Ideally, we should provide appropriate spacing between lines and paragraphs, and we shouldn’t be requiring the user to scroll horizontally to read a line of text on a full-screen window.” – WCAG 1.4.8
  • “Users must be able to resize text without being forced to scroll both horizontally and vertically to read that content.”-WCAG 1.4.10

As a result, I think relative length units should be used for a responsive outcome so that users can control their web browsing experience even when they change their settings on factors like zooming. Fixed pixel values for attributes like font size and line height don’t scale with browser adjustments making it difficult for the user. In general using relative CSS units helps developers maintain their layouts without negatively impacting user’s needs.

Browser Interpretation of Width

Default browser width is set to 16pxs. Pixel is an absolute measurement, while percent and EM are relative measurements. Percent depends on its parent font size and EM is relative to the current size of the element.

The standard conversion formula is 16px = 100% = 1em. The browser interprets units from the cascading style sheets doing calculations based on this formula.

For example: 2em means 2 times the size of the current size. So, if the current size of body is 16 pixels, then 150% will be 24 pixels based on the calculation (1.5 * 16), and 2em will be 32 pixels (16 * 2). And just as 200px converts to 12.5em.

New Feauters in CSS3

  1. Some new selectors were introduced. These new selectors allow users to access DOM elements through their attributes making styling easier. For example the most useful attributes for selectors are:

    • [attr*=val] Â–- matches a DOM element with the attribute attr and a value containing the substring val

  2. Rounded Corners were introduced through the border radius property removing the hustle of adjusting height, width and posititioning of elements the developer had to go through. Rounder corners can be applied to different elements like text and images in html.

  3. The new Box Shadow and text shadow effects of css3 enable a shadow effect to be dropped around an element through a single line of code of css3 which was previously done using a repeated image around the element.

  4. The color keyword list in CSS3 has been extended to include additional 147 keywords and also the ability to use semi-transparent colors through options like HSL, HSLA and RGBA.

  5. Linear Gradients and Radial Gradients were also introduced to CSS3 for blending colors in differents angles for styling purposes.

  6. New pseudo classes of selectors were introduced in CSS3. Some of these include:
    • :root
      The root element of the document.
    • :nth-child(n)
      Use this to match exact child elements or use variables to get alternating matches.
    • :nth-last-child(n)
      Match exact child elements counting up from the last one.
    • :nth-of-type(n)
      Match sibling elements with the same name before it in the document tree.
    • :nth-last-of-type(n)
      Match sibling elements with the same name counting up from the bottom.
    • :last-child
      Match the last child element of the parent.
    • :first-of-type
      Match the first sibling element of that type.
    • :last-of-type
      Match the last sibling element of that type.
    • :only-child
      Match the element that is the only child of its parent.
    • :only-of-type
      Match the element that is the only one of its type.
    • :empty
      Match the element that has no children (including text nodes).
    • :target
      Match an element that is the target of the referring URI.
    • :enabled
      Match the element when it's enabled.
    • :disabled
      Match the element when it's disabled.
    • :checked
      Match the element when it's checked (radio button or checkbox).
    • :not(s)
      Match when the element does not match the simple selectors.
  7. Multiple Background Images were also introduced to CSS3 to be layered over one another. The first layer being the one closer to the user and the others following behind. If there is a background color it is layered behind all the images.
  8. New background properties include:
    • background-clip: defines how background image should be clipped, the default being the border box with options of being changed to the padding box or the content box.
    • background-origin: determines whether the background should be placed in the padding box, the border box, or the content box.
    • background-size: indicates the size of the background image. It allows you to stretch smaller images to fit the page.
  9. New border properties include:
    • border-radius : to create rounded corners on your borders.
    • border-image-source : to specify the image source file
    • border-image-slice : to represent the inward offset from image edges.
    • border-image-width :defines the value of the width for your border image.
    • border-image-outset :to specifiy the amount that the border image area extends beyond the border box.
    • border-image-stretchefines how the sides and middle parts of the border image should be tiled or scaled.
